    Principles for consolidation & Elimination

    • Rules for selecting the positions to report and to eliminate depending on the consolidation level:
      • BankOffice:
        • Report positions of booking centers (core_positions.BookingCenterId) in (CH,LI) (list_booking_centers.OfficeCountry) which have the following type of office (list_booking_centers.OfficeType):
          • FinancialHolding
          • HeadOffice
          • Branch (Sucursale)
          • RepresentationOffice (part of Branch)
          • Agency (part of Branch)
        • Eliminate positions where the counterparty is a booking center (core_positions.CounterpartyBookingCenterId) with the same characteristics as above
      • ParentCompany
        • Report positions of booking centers (core_positions.BookingCenterId) in any country (list_booking_centers.OfficeCountry) which have the following type of office (list_booking_centers.OfficeType):
          • FinancialHolding
          • HeadOffice
          • Branch (Sucursale)
          • RepresentationOffice (part of Branch)
          • Agency (part of Branch)
        • Eliminate positions where the counterparty is a booking center (core_positions.CounterpartyBookingCenterId) with the same characteristics as above
      • Group
        • Report positions of all booking centers (core_positions.BookingCenterId) in any country (list_booking_centers.OfficeCountry) which have any type of office (list_booking_centers.OfficeType) → Subsidiary (Filliale) added compared to ParentCompany
        • Eliminate all positions where the counterparty is a booking center (core_positions.CounterpartyBookingCenterId)

    Workflow for RT01 input

    1. User imports RT01
    2. Mercury fills table list_reporting_entities → one line for each unique combination of Entity and EntityType = "CH"
      • Id → Entity
      • SNBCode empty
      • ConsoLevelBankOffice, ConsoLevelParentCompany, ConsoLevelGroup set to TRUE by default
    3. Mercury fills table list_booking_centers → one line for each Entity
      • Id (unique) → Entity
      • OfficeType / OfficeCountry → EntityType through mapping (see document Interface_RT01-QDL)
      • OfficeCountry → CounterpartyDomicile (if not inferred from EntityType)
      • By default, fill the field OwnerId with the Id of of the first entry of list_reporting_entities
    4. Mercury fills table list_reporting_sets
      • ReportingEntity (pk) → list_reporting_entities.Id
      • ConsolidatedBookingCenterId (pk) → list_booking_centers.Id which have ReportingEntity as the OwnerId in list_booking_centers
      • ConsolidatedBookingCenterOfficeType:
        • HeadOffice when ReportingEntity = ConsolidatedBookingCenterId
        • Branch when list_booking_centers.OfficeType = Branch
        • Subsidiary when list_booking_centers.OfficeType = Subsidiary
    5. User amends tables list_reporting_entities, list_booking_centers, list_reporting_sets
    6. User launch calculation on a load collection and must choose the ReportingEntity for that load collection if empty
      • Each load collection has its set of reports

    Filtering in the reports:

    Consolidation level BankOffice

    
select * from core_positions cp inner join core_positions_calc cpc
on cp.Id = cpc.Id and cp.LoadId = cpc.LoadId
where cp.BookingCenterId in (
    select ConsolidatedBookingCenterId from
    list_reporting_sets lrs inner join core_load_collections clc
    on lrs.ReportingEntityId = clc.ReportingEntityId
    where lrs.OfficeType in ('FinancialHolding','HeadOffice','Branch','Agency','RepresentationOffice')
    and lrs.OfficeCountry in ('CH','LI')
    and clc.LoadCollectionId in
        (select core_loads_loads_collection.LoadCollectionsLoadCollectionId from 
        core_loads_loads_collection cllc inner join core_positions cp
        on cllc.LoadsLoadId = core_positions.LoadId)
and cp.CounterpartyBookingCenterId not in (
    select ConsolidatedBookingCenterId from
    list_reporting_sets lrs inner join core_load_collections clc
    on lrs.ReportingEntityId = clc.ReportingEntityId
    where lrs.OfficeType in ('FinancialHolding','HeadOffice','Branch','Agency','RepresentationOffice')
    and lrs.OfficeCountry in ('CH','LI')
    and clc.LoadCollectionId in
        (select core_loads_loads_collection.LoadCollectionsLoadCollectionId from 
        core_loads_loads_collection cllc inner join core_positions cp
        on cllc.LoadsLoadId = core_positions.LoadId)
